# 三、硬件配置分级要求

软件启动自动全维度硬件检测（耗时≤10秒），并生成可导出的硬件报告（hardware-report.json）。本产品 V1.0 仅提供“最佳效果”生成链路（零样本音色克隆 + 离线ASR参考文本 + 图片口播生成 + MuseTalk口型精修 + 画质增强 + 强制字幕对齐 + 1080P合成，默认30fps）。

核心原则：不提供低配降级生成；硬件不达标或运行前资源不足（Preflight不通过）将直接禁止开始生成，并提示升级配置/释放资源。

### 3.0 关键说明（最佳效果默认策略）

- 仅支持 NVIDIA GPU（CUDA），不支持 AMD GPU。
- 最佳效果模式不支持纯 CPU 推理。
- 批量制作支持一次性导入≤100条，但严格按队列串行生成（同一时刻仅1个作业）。

## 3.1 最低准入配置（最佳效果·30fps 可生成）

适用人群：在满足硬门槛的前提下运行“最佳效果”全链路，但对生成效率容忍度较高。

- CPU：必须支持 AVX2；建议 ≥6核12线程
- 内存（硬门槛）：RAM 总量 ≥16GB；运行前可用内存（Preflight Available RAM）≥10GB
- 虚拟内存（硬门槛）：Pagefile 必须启用，且建议大小 ≥32GB（未启用或过小直接拒绝生成）
- 显卡（硬门槛）：NVIDIA 且 VRAM 总量 ≥8GB；运行前可用显存（Preflight Free VRAM）≥6GB
- 显卡驱动（硬门槛）：建议 NVIDIA Driver ≥531.xx（低于则拒绝生成并提示升级驱动）
- CUDA（硬门槛）：CUDA 自检必须通过
- 磁盘（硬门槛）：临时目录与作品目录所在分区必须为 SSD；可用空间 ≥100GB；必须具备写权限

系统强制策略：
- 仅允许单任务串行（同一时刻仅1个作业）
- 默认输出 1080P / 30fps
- “极致模式（60fps插帧）”开关置灰（由 3.3 规则控制）

## 3.2 推荐稳定配置（最佳效果·30fps 默认体验最佳）

适用人群：追求最佳效果同时希望更稳、更少失败、更少受后台占用影响。

- CPU：建议 ≥8核16线程
- 内存（推荐）：RAM ≥32GB
- 显卡（推荐）：NVIDIA，VRAM ≥12GB
- 磁盘：NVMe SSD 推荐；可用空间建议 ≥150GB

默认开启：离线ASR参考文本、MuseTalk口型精修、强制字幕对齐、画质增强与时序稳定。

## 3.3 旗舰极致配置（极致模式·60fps插帧可用）

适用人群：对观感上限要求极高，开启 60fps 插帧与更激进的增强参数。

- 内存（硬门槛）：RAM ≥64GB
- 显卡（硬门槛）：NVIDIA，VRAM ≥16GB
- 磁盘：NVMe SSD 强烈建议；临时空间建议 ≥150GB

开启极致模式后，允许输出 1080P / 60fps（插帧）。

## 3.4 最佳效果模式硬件检测与硬门槛（不满足则禁止生成）

### 3.4.1 启动检测项
- OS：Windows 10/11 版本、64位
- CPU：型号、物理核/线程、是否支持 AVX2
- RAM：总量、可用内存（Available）
- Pagefile：是否启用、当前大小
- GPU：是否 NVIDIA、VRAM 总量、驱动版本、CUDA 可用性、当前可用显存
- 磁盘：临时目录所在盘是否 SSD/NVMe、可用空间、写权限

### 3.4.2 运行前 Preflight（点击“生成视频/开始批量”时必做）
- Free VRAM ≥6GB
- Available RAM ≥10GB
- 临时目录可用空间 ≥100GB
- CUDA 自检通过

Preflight 任一不通过：直接禁止生成（不入队、不创建作业），并提示当前值/要求值/升级建议。

### 3.4.3 硬件失败码（用于 UI 提示与验收）
- HW_OS_UNSUPPORTED
- HW_CPU_NO_AVX2
- HW_RAM_INSUFFICIENT_TOTAL（总内存不足16GB）
- HW_RAM_INSUFFICIENT_AVAILABLE（可用内存不足10GB）
- HW_PAGEFILE_DISABLED_OR_SMALL
- HW_GPU_NOT_NVIDIA
- HW_VRAM_INSUFFICIENT_TOTAL（总显存不足8GB）
- HW_VRAM_INSUFFICIENT_FREE（可用显存不足6GB）
- HW_DRIVER_TOO_OLD
- HW_CUDA_UNAVAILABLE
- HW_DISK_NOT_SSD
- HW_DISK_INSUFFICIENT_SPACE
- HW_NO_WRITE_PERMISSION

### 3.4.4 硬件检测报告导出
导出 hardware-report.json，包含 app_version、模型版本、CPU/RAM/Pagefile/GPU/磁盘详细数据、pass/fail 结果、失败原因列表（含 current/required/suggestion）。

# 四、产品整体架构（含技术架构联动）

## 4.1 界面布局

采用「左侧固定功能菜单+右侧核心内容区」Windows桌面端经典布局，左侧为8大核心模块入口，右侧为对应模块的操作界面，整体风格简洁明了、无冗余，符合Windows用户操作习惯，支持浅色/深色主题一键切换，切换后实时生效；新增单张图片生成数字人入口、自定义音色上传入口，布局贴合原有操作逻辑，不增加用户学习成本。

## 4.2 核心模块划分（8大模块，独立联动，围绕视频制作核心场景）

1. 首页模块：软件唯一入口，提供核心功能快速跳转、最近操作记录、新手教程引导，实时展示硬件状态与GPU加速情况。

2. 单条制作模块：核心功能模块，采用5步步骤化引导+右侧实时预览布局，实现单条数字人口播视频全流程制作，集成单张图片生成数字人、自定义音色选择功能。

3. 批量制作模块：支持一次性导入≤100条文案形成任务队列，统一配置参数（音色样本/数字人图片/字幕样式/BGM 等），严格按队列串行生成（同一时刻仅1个作业）；批量开始前与每条开跑前执行 Preflight（可用显存/可用内存/磁盘空间/CUDA），不通过直接拦截并提示升级配置/释放资源。

4. 数字人管理模块：管理官方数字人（内置形象）与用户“图片数字人”（单张图片生成的头像/遮罩/关键点等资料），支持预览、收藏、使用、删除、重新生成（重新上传图片），用于后续 SadTalker+MuseTalk 生成口播视频。

5. 音色管理模块：管理内置音色与用户“零样本克隆音色”（上传人声样本生成 speaker_profile），支持试听、收藏、使用、上传/重建、删除与空间清理；不支持导入 PT 等模型文件。

6. 作品库模块：本地统一管理所有生成的口播视频，支持查看、播放、编辑、删除、批量操作，重新编辑可精准加载原始配置（含数字人、音色参数）。

7. 设置模块：配置软件运行参数、修改模型/视频存储路径（全英文路径）、优化性能设置、查看硬件信息、清理缓存、重置所有设置。

8. 帮助与反馈模块：提供本地离线使用教程、常见问题（FAQ）解答、客服入口、问题反馈通道，含单张图片生成数字人、自定义音色上传的常见问题解决方案。

## 4.3 核心技术流程（最佳效果链路：全离线本地完成）

1. 文案处理：用户输入/导入文案 → 本地断句/口语化优化/长度校验 → 生成可播报脚本片段（sentence list）。

2. 音色建档（零样本克隆）：用户上传人声样本（WAV/MP3）→ VAD 切分/降噪（可选）/响度归一化 → 离线 ASR 生成参考文本 prompt_text（Whisper，可编辑）→ 生成并缓存 speaker_profile。

3. 语音合成（TTS）：CosyVoice3-0.5B + speaker_profile + prompt_text + 情感/语速/音量参数 → 生成 WAV → 音频后处理（重采样、限幅/响度一致性）→ 得到 final_audio.wav。

4. 图片口播基视频：选择官方形象或上传单张正面清晰照片 → 人脸检测/对齐/关键点 → SadTalker 生成 1080P/30fps 基视频 base.mp4。

5. 口型精修：MuseTalk 以 final_audio.wav 对 base.mp4 的口周 ROI 做口型精修 → 输出 lipsync_refine.mp4（减少漂移、提升自然度）。

6. 画质增强与时序稳定：人脸修复/超分（GFPGAN/Real-ESRGAN）、去闪烁、姿态/关键点平滑 → 输出 enhanced.mp4。

7. 字幕强制对齐：对 final_audio.wav 做离线 ASR（Whisper）→ WhisperX/MFA 生成字/词级时间轴 → 生成 ASS 字幕脚本（支持样式配置）。

8. 合成输出：FFmpeg 合成（视频 + 字幕 burn-in + BGM 混音）→ 编码输出 final.mp4 → 生成缩略图与 meta.json → 写入作品库索引。

说明：涉及推理/合成的阶段在开始前必须通过硬门槛与 Preflight（可用显存/可用内存/磁盘空间/CUDA），不通过直接拦截并提示升级配置/释放资源。

## 4.4 核心技术架构（分层设计，支撑功能落地）

整体技术架构分为5层，自上而下实现「交互入口→适配处理→核心引擎→模型支撑→存储管理」全链路本地化，联动所有功能模块，确保单张图片生成数字人、自定义音色上传等核心功能可落地，各层职责清晰、低耦合。

1. 交互适配层：承接所有模块交互需求，负责单张图片上传、自定义音色上传等操作的前端适配，以及不同硬件配置的动态适配，传递用户指令至核心引擎层，反馈处理结果至前端。

2. 核心引擎层：核心技术支撑，包含口型同步引擎（Wav2Lip轻量优化版）、数字人渲染引擎（Unity轻量版）、自定义数字人适配引擎、单张图片生成数字人引擎（SadTalker轻量化版）、自定义音色适配引擎、视频合成引擎（FFmpeg）。

3. 模型支撑层：内置核心模型，包含CosyVoice3-0.5B语音模型、Wav2Lip口型驱动模型、SadTalker图片转数字人模型、官方数字人基础模型、渲染优化模型，支持自定义模型本地加载与校验。

4. 格式解析层：负责数字人（FBX/GLB/OBJ/图片）、音色相关文件的格式解析与转换，将用户上传的文件转换为引擎可识别的专用格式，完成轻量化优化。

5. 存储管理层：负责所有模型、素材、作品的本地存储（全英文路径），支持模型缓存清理、冗余文件删除，采用索引机制提升加载速度，确保本地读取延迟≤1秒。

## 5.2 单条制作模块（核心功能模块，含核心技术落地）

### 5.2.1 功能描述

采用“左侧5步步骤化引导+右侧实时预览区”的布局，实现单条数字人口播视频的全流程制作，步骤间可自由回退，所有配置修改1秒内同步至预览区；集成单张图片生成数字人、自定义音色选择功能，技术上适配本地离线推理，兼顾易用性与性能。

### 5.2.2 界面布局

- 左侧：固定5步制作流程，依次为「文案输入→语音选择→数字人选择（含单张图片生成）→视频设置→生成视频」，当前操作步骤高亮显示，已完成步骤标注“√”，支持点击任意已完成步骤回退，回退后保留所有配置。

- 右侧：实时预览区，占界面70%以上空间，同步展示当前步骤的配置效果（含图片生成的数字人预览、自定义音色语音预览），支持语音播放、视频预览，所有参数修改1秒内更新预览效果。

### 5.2.3 分步功能详情（含技术解决方案）

#### 步骤1：文案输入（核心：文案处理、校验、优化）

2. 音色核心操作：试听、选择、收藏，试听延迟≤1秒，语音清晰无杂音；技术适配：
        

    - 内置音色：基于CosyVoice3-0.5B模型衍生，本地推理生成语音，支持CPU/GPU自适应加载，GPU加速时语音合成速度提升3-5倍；

    - 自定义音色：用户上传后，本地完成格式解析、模型适配，生成可推理模型，适配后支持与内置音色同等操作，语音合成时调用本地自定义模型，无云端依赖。

3. 语音参数调节：语速（0.5-2.0）、音量（0-200%）、情感强度（0-100%），调节后实时更新预览效果；技术适配：参数调节实时传递至语音合成引擎，推理参数动态调整，无需重新加载模型，响应延迟≤0.5秒。

    - 适配流程：本地格式解析层校验文件合法性 → 转换为引擎可识别格式 → 音色模型轻量化优化（体积压缩30%）→ 绑定语音合成引擎 → 生成试听样本 → 完成适配；

    - 技术选型：基于CosyVoice3-0.5B模型微调接口，本地完成模型适配，无需联网，适配耗时基础档≤60秒、流畅档≤30秒、高效档≤15秒；

5. 操作按钮：上一步（保留配置回退）、下一步（仅选择音色后可点击）；技术适配：跳转时自动保存音色参数，与后续数字人口型同步模块联动，确保语音与口型精准匹配。

#### 步骤3：数字人选择（核心：支持单张图片生成、自定义上传、官方数字人，含完整技术方案）

1. 数字人分类展示：按“我的收藏、男主播、女主播、职场、亲和、专业、自定义数字人、图片生成数字人”划分，每个数字人卡片为高清预览图，标注风格；技术适配：数字人列表从本地英文路径读取，采用索引加载，预览图缓存至本地，加载延迟≤1秒。

2. 数字人核心操作：预览、收藏、立即使用，预览时展示动作小样与口型同步效果，口型同步误差≤0.1秒；技术适配：预览时调用本地渲染引擎（Unity轻量版），实时渲染数字人动作与口型，GPU加速时预览无卡顿，纯CPU推理时优化渲染帧率（≥15fps）。

3. 单张图片生成数字人（核心功能，含完整技术解决方案）：
        

    - 入口：数字人分类栏右上角“上传单张图片生成数字人”按钮，点击弹出上传弹窗，引导用户上传正面清晰照片；

    - 支持格式与要求：JPG/PNG格式，单文件≤20MB，要求正面清晰人脸、无遮挡、光线均匀，无侧脸/模糊/多人脸；

    - 技术选型：采用SadTalker轻量化版模型（内置至安装包，体积≤80MB），结合Wav2Lip口型驱动模型，本地离线推理，无需建模/多角度图片/视频；

    - 生成流程（全本地完成）：
                1. 用户上传单张图片 → 本地人脸检测引擎（MTCNN轻量版）检测人脸，无人脸则提示“请上传清晰正面人脸照片”；
                2. 人脸关键点定位（提取68个人脸关键点）→ 生成3D人脸模型，绑定基础动作（坐姿/站姿）；
                3. 口型驱动绑定（匹配46种基础口型，适配语音驱动）→ 生成可动数字人模型；
                4. 自动生成预览小样（3秒动作+语音口播预览）→ 完成生成，同步添加至“图片生成数字人”分类，支持直接选择使用；
              

    - 性能指标：生成耗时基础档≤60秒、流畅档≤30秒、高效档≤15秒，生成的数字人支持口型同步、动作联动，视觉自然，无明显畸变；

    - 异常处理：图片模糊、人脸遮挡、光线过暗时，提示“图片不符合要求，请重新上传”；生成超时（基础档＞120秒、流畅档＞60秒、高效档＞30秒），自动终止推理，提示“生成超时，请检查硬件配置或重新上传图片”；

    - 补充功能：支持重新上传图片生成数字人，覆盖原有模型；生成的数字人支持收藏、删除，删除时同步删除本地模型文件与预览素材。

4. 自定义数字人上传入口：数字人分类栏右上角“上传自定义数字人”按钮，点击跳转至数字人管理模块上传页面，上传完成后自动返回本步骤；技术解决方案：
        

    - 支持格式：FBX、GLB、OBJ，单文件大小基础档≤50MB，流畅/高效档≤100MB，要求为标准人体模型（含完整面部骨骼）；

    - 适配流程：本地格式解析层校验文件完整性 → 转换为引擎可识别的.asset格式 → 模型轻量化优化（面数≤5万面）→ 骨骼绑定与口型适配 → 生成预览小样 → 完成适配；

    - 技术选型：基于Unity模型导入接口，自定义适配引擎，本地完成骨骼绑定与口型驱动适配，无需联网；

    - 异常处理：模型骨骼缺失、格式不兼容时，提示具体原因并给出解决方案，适配失败自动清理临时文件。

5. 操作按钮：上一步（保留配置回退）、下一步（仅选择数字人后可点击）；技术适配：跳转时自动保存数字人配置，与语音参数联动，确保后续口型同步精准。

#### 步骤4：视频设置（核心：视觉、音频参数自定义，实时预览）

1. 背景设置：支持内置背景和自定义背景（上传本地图片），点击选择后1秒内更新预览效果；技术适配：背景图片加载调用本地渲染引擎，自动适配视频比例，自定义背景上传后缓存至本地英文路径，支持后续复用。

2. 生成操作按钮：暂停/继续、取消，点击后1秒内响应，暂停后可继续，取消后提示确认，避免误操作；技术适配：暂停时，各引擎暂停推理，保留当前生成进度，继续时从中断步骤恢复，无需重新生成；取消时，自动清理临时生成文件，释放硬盘空间。

    - 日志记录：失败日志自动记录至本地英文路径，支持用户导出日志，便于排查问题。

## 5.3 批量制作模块（最佳效果：100条导入 + 队列串行执行）

### 5.3.1 功能定位

面向矩阵号/企业批量生产场景：允许用户一次性导入最多 100 条文案形成任务队列。为了保证“最佳效果链路”（零样本音色克隆 + 离线ASR参考文本 + 图片口播生成 + MuseTalk口型精修 + 画质增强 + 强制字幕对齐 + 1080P 合成）稳定一致，队列内任务必须严格串行执行（同一时刻仅运行 1 个作业）。

### 5.3.2 核心规则（验收点）

1. 导入上限：单次最多导入 100 条（粘贴多行/导入 TXT，一行一条）。

2. 执行方式：队列严格串行：上一条完整跑完（含合成输出）后才开始下一条；禁止并发生成。

3. 硬件准入：开始批量前必须通过“最佳效果模式硬门槛 + Preflight（可用显存/可用内存/磁盘空间）”，不通过则直接禁止开始并提示升级配置/释放资源。

4. 参数一致性：同一批次使用同一套参数（音色样本/数字人图片/视频设置/字幕样式/BGM 等）。若队列执行中修改参数，仅对“未开始”的任务生效（或要求先暂停队列再应用变更，产品需固定一种交互并保持一致）。

5. 失败策略（推荐默认）：单条失败记录原因并自动跳过，继续处理后续任务；队列结束输出汇总报告，并支持“一键重试失败项”。

### 5.3.3 队列与作业模型（技术契约）

- 队列状态：idle / running / paused / cancelling / finished
- 单条作业状态：queued / running / succeeded / failed / cancelled
- 单条作业阶段（用于进度展示与日志定位）：
  text -> voice_profile -> asr_prompt -> tts -> avatar_base -> lipsync_refine -> enhance_stabilize -> subtitle_align -> compose

### 5.3.4 进度展示（验收点）

- 总进度：当前条数/总条数（例如 23/100）
- 当前条阶段进度：显示阶段名称 + 百分比（例如“口型精修 62%”）
- 关键提示：显示是否启用“极致模式 60fps”、是否通过 Preflight、是否发生自动重试（若实现）

### 5.3.5 控制能力（验收点）

- 暂停队列：在安全点暂停（优先在阶段边界）；暂停后不启动下一条
- 继续队列：从当前条最近 checkpoint 或阶段起点恢复（需明确语义）
- 取消队列：默认取消未开始任务；当前任务提供二次确认“立即取消并清理临时文件”
- 重试失败项：仅把失败项重新入队（保持原参数快照，避免重试时参数漂移）

### 5.3.6 Preflight 策略（保证批量稳定性）

- 批量开始前：做一次全局 Preflight（不通过直接拦截）
- 每条开跑前：做一次快速 Preflight（防止中途被其他程序占用导致失败）
- Preflight 不通过：该条不开始，提示“当前资源不足/被占用”，并提供“重试/跳过/取消队列”选项（默认策略需明确）

### 5.3.7 输出与命名（建议写清，避免冲突）

- 输出目录：统一为用户设置的作品目录（必须可写、SSD）
- 命名规则：支持前缀 + 自增序号 + 时间戳（防重名）
- 每条输出：final.mp4 + thumbnail + meta.json + log_path
- 批量汇总报告：成功数/失败数/失败原因列表/日志导出入口

### 5.3.8 批量性能优化（不改变“串行”原则）

在不并发重模型的前提下允许轻量预处理流水线：当第 N 条进入合成阶段（FFmpeg）时，可提前对第 N+1 条做文本规范化/字幕脚本生成（轻量 CPU 任务），但 ASR/TTS/生成/口型精修/增强/对齐 等重阶段仍保持串行且互斥加载。

1. 音色分类管理：按“我的收藏、男声、女声、情感音、方言音、自定义音色”分类展示，支持排序；技术适配：分类索引本地维护，从指定英文路径读取音色模型信息，加载延迟≤1秒。

2. 音色核心操作：
        

    - 试听：点击试听按钮，1秒内播放语音样本，清晰无杂音；技术适配：试听样本本地缓存，内置音色复用默认样本，自定义音色复用上传时生成的样本，避免重复推理；

    - 收藏/取消收藏：同步更新分类列表，收藏状态本地持久化；

    - 删除：删除音色时，同步删除本地模型文件、试听样本、索引记录；技术适配：删除前弹出确认提示，删除后自动更新索引，清理冗余缓存；

    - 重新加载：支持重新加载音色模型，解决模型加载异常问题；技术适配：重新加载时，重新校验模型完整性，从本地英文路径读取最新模型文件，加载失败提示解决方案。

3. 自定义音色上传与适配：支持上传语音样本/模型文件，本地完成适配，适配后支持所有音色操作；技术解决方案与步骤2一致，补充适配后模型管理功能，支持查看适配参数、重新适配。

4. 音色模型优化：支持清理音色缓存，释放硬盘空间；技术适配：缓存文件存储至指定英文路径，清理时保留核心模型文件，仅删除试听样本、临时适配文件，不影响音色使用。

5. 异常处理：音色模型缺失、损坏、适配失败时，提示具体原因并给出解决方案；技术适配：内置音色自动修复，自定义音色提示重新上传/适配，语音合成时音色模型加载失败，自动切换至默认内置音色，确保用户可正常操作。
