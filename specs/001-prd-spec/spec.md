# Feature Specification: 智影口播 · AI数字人视频助手（Windows版）V1.0

**Feature Branch**: `001-prd-spec`
**Created**: 2026-02-23
**Status**: Draft
**Input**: 基于《智影口播·AI数字人视频助手（Windows版）PRD文档（V1.0 完善版）》整理生成

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 新手完成首条口播视频（Priority: P1）

用户首次打开软件，使用内置数字人与内置音色，按 5 步向导输入文案并生成一条 1080P 口播视频，生成完成后可立即播放、打开文件位置，并在作品库中看到记录。

**Why this priority**: 这是软件的核心价值闭环（文字 → 视频），也是新手是否能“马上用起来”的关键。

**Independent Test**: 仅实现“单条制作向导 + 生成 + 本地保存 + 作品库记录”，即可独立演示与验收。

**Acceptance Scenarios**:

1. **Given** 用户已安装并打开软件且设备满足最低要求，**When** 用户在单条制作中输入不少于 10 字文案并完成向导配置后点击生成，**Then** 系统应展示清晰的生成进度与当前阶段，并在生成成功后提供播放与打开文件位置的入口。
2. **Given** 用户已生成成功一条视频，**When** 用户进入作品库查看最近作品，**Then** 应能看到该作品卡片（含缩略图/名称/时间），并可进行播放与删除操作。
3. **Given** 用户未联网，**When** 用户完成单条制作并生成视频，**Then** 生成流程应不依赖联网即可完成，且作品与素材均仅存储于本地。

---

### User Story 2 - 批量生成多条口播视频（Priority: P2）

用户导入或粘贴多条文案（每行一条），统一选择数字人/音色/视频设置后，发起批量生成；系统按规则生成并提供整体与单条进度，完成后汇总成功与失败条目。

**Why this priority**: 这是面向矩阵号/企业批量内容生产的核心增效能力。

**Independent Test**: 在不实现“自定义素材上传”的前提下，仅靠“批量文案 + 统一配置 + 串行/按能力限制的批量生成 + 汇总结果”即可完成验收。

**Acceptance Scenarios**:

1. **Given** 用户输入多行文案并选择统一参数，**When** 用户点击开始批量生成，**Then** 系统应展示整体进度与当前正在生成的条目，并支持暂停/继续/取消。
2. **Given** 批量任务包含多条文案，**When** 其中某条生成失败，**Then** 系统应记录失败原因并继续处理后续条目，最终展示成功/失败统计与失败明细。

---

### User Story 3 - 单张图片生成数字人并用于口播（Priority: P2）

用户上传一张符合要求的正面清晰照片，系统在本地生成可用于口播的数字人，并将其加入数字人库；用户随后在单条/批量制作中选择该数字人生成视频。

**Why this priority**: “单张图片生成数字人”是产品差异化能力，直接提升个性化与可用场景。

**Independent Test**: 仅实现“图片校验 → 生成数字人 → 数字人库可见 → 在单条制作中可选择并生成视频”，即可独立验收。

**Acceptance Scenarios**:

1. **Given** 用户上传 JPG/PNG 图片且满足清晰正面人脸要求，**When** 用户确认生成数字人，**Then** 系统应在与设备等级匹配的目标时间内完成生成并展示预览结果（入门档≤60秒、推荐档≤30秒、专业档≤15秒），生成完成后该数字人应出现在数字人列表中并可被选用。
2. **Given** 用户上传的图片不满足要求（如模糊、非正面、多人脸或无人脸），**When** 用户尝试生成数字人，**Then** 系统应给出明确的失败原因与改进建议，并不应产生残留的无效数字人记录。

---

### User Story 4 - 上传自定义音色并使用（Priority: P3）

用户在音色管理中上传自定义音色素材（样本或模型文件），系统完成本地适配后，在单条/批量制作中可像内置音色一样试听与选择。

**Why this priority**: 自定义音色提升品牌一致性与创作差异化，但可在核心闭环稳定后再落地。

**Independent Test**: 仅实现“上传 → 校验 → 适配 → 生成试听样本 → 在音色列表可选择”即可独立验收。

**Acceptance Scenarios**:

1. **Given** 用户上传的音色素材格式与大小符合要求，**When** 系统完成适配，**Then** 用户应能在音色列表中看到该音色并可试听与选用。
2. **Given** 用户上传的音色素材缺失/损坏/不兼容，**When** 系统尝试适配，**Then** 系统应提示明确原因并清理临时文件，不影响用户继续使用内置音色。

---

### User Story 5 - 本地作品与素材管理（Priority: P3）

用户在作品库中管理已生成的视频（查看、搜索/排序、播放、删除、重新编辑），并在设置中管理本地存储路径与缓存清理。

**Why this priority**: 作品沉淀与可持续使用决定长期留存；同时要避免本地空间被缓存占满。

**Independent Test**: 仅实现“作品库列表 + 详情/播放 + 删除 + 基础路径设置 + 缓存清理”即可独立验收。

**Acceptance Scenarios**:

1. **Given** 用户已有多条生成作品，**When** 用户在作品库中按名称搜索或按时间排序，**Then** 系统应在 1 秒内返回结果并保持 UI 可操作（在作品数量达到 500 条时仍满足该目标）。
2. **Given** 用户删除某个作品，**When** 用户确认删除，**Then** 系统应同时移除作品库记录与对应本地视频文件，并释放磁盘空间。

### Edge Cases

- 文案不足最小字数、超过最大字数或包含不支持字符时如何提示与拦截？
- 文案优化失败或超时如何回退到原文？
- 生成过程中磁盘空间不足时如何处理（中止、提示、清理临时文件）？
- 用户选择的保存路径不可写/不存在/包含非英文字符时如何处理？
- 设备无可用 GPU 时如何提示并自动采用可用的推理策略？
- 批量生成时用户关闭软件/异常退出后如何恢复或提示重新开始？
- 暂停/继续/取消在不同阶段触发时的行为一致性如何保证？
- 自定义素材上传后立即删除、或适配失败重试时如何防止残留与重复记录？

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 支持 Windows 10/11（64 位）平台运行，并在启动时检查关键环境（如磁盘类型与可用空间）并给出可操作的整改提示。
- **FR-002**: 系统 MUST 在启动后完成硬件检测并为用户呈现性能等级与能力边界（如批量条数上限、素材上传大小上限等）。
- **FR-003**: 系统 MUST 提供“左侧步骤导航 + 右侧预览区”的单条制作向导流程，包含文案输入、语音选择、数字人选择、视频设置、生成视频 5 个步骤。
- **FR-004**: 用户 MUST 能在单条制作的任意已完成步骤间回退，并且系统 MUST 保留已配置内容不丢失。
- **FR-005**: 系统 MUST 对文案进行本地校验（最小字数、最大字数等）并在不满足条件时阻止进入下一步。
- **FR-006**: 系统 MUST 提供本地文案优化能力（断句、口语化等），并在优化后允许用户一键还原到优化前版本。
- **FR-007**: 系统 MUST 提供音色浏览、选择与试听能力，且试听操作 MUST 给出清晰的可感知反馈（加载/播放/失败原因）。
- **FR-008**: 系统 MUST 提供语音参数调节能力（如语速、音量、情感强度），并使用户能够在生成前预览调节效果。
- **FR-009**: 系统 MUST 提供数字人浏览、选择与预览能力，并支持收藏/取消收藏。
- **FR-010**: 系统 MUST 支持“单张图片生成数字人”的入口与流程，包括图片格式/大小校验、清晰正面人脸要求提示、生成结果预览、保存到数字人库。
- **FR-011**: 系统 MUST 支持用户上传自定义数字人素材并完成适配，使其可在数字人库中被预览与选用。
- **FR-012**: 系统 MUST 支持用户上传自定义音色素材并完成适配，使其可在音色库中被试听与选用。
- **FR-013**: 系统 MUST 支持视频背景设置（内置与自定义），且用户修改后预览区 MUST 在 1 秒内更新或给出明确的加载反馈。
- **FR-014**: 系统 MUST 支持视频比例切换（至少 9:16 与 16:9），且输出分辨率 MUST 为 1080P。
- **FR-015**: 系统 MUST 默认提供字幕并允许用户开关与样式配置，生成结果 MUST 包含字幕效果；除试用模式水印外，不应出现额外水印。
- **FR-016**: 系统 MUST 支持 BGM 关闭/启用，并允许用户选择内置或自定义 BGM 且可独立调节音量。
- **FR-017**: 系统 MUST 在生成过程中展示可视化进度与阶段信息，并允许用户暂停/继续/取消。
- **FR-018**: 当用户取消或生成失败时，系统 MUST 清理临时文件并避免产生不可见但占用空间的残留。
- **FR-019**: 系统 MUST 将生成的视频保存到用户指定的本地路径，并在生成成功后提供“播放”“打开文件位置”“重新生成”“制作下一条”等快捷操作。
- **FR-020**: 系统 MUST 提供作品库用于本地作品管理，支持查看、播放、删除、批量操作，以及根据作品名称/时间等条件的筛选或排序。
- **FR-021**: 系统 MUST 支持“重新编辑”能力：用户从作品库打开某条作品后，可加载其生成时的配置并进入单条制作流程继续编辑。
- **FR-022**: 系统 MUST 提供设置入口，允许用户修改模型/素材/作品的本地存储路径，并在路径不可用时给出明确提示；系统 MUST 持久化设置并在重启后保持一致。
- **FR-023**: 系统 MUST 提供缓存清理能力，清理后 MUST 告知释放空间大小，并保证清理不会误删用户作品文件。
- **FR-024**: 系统 MUST 提供帮助与反馈模块，包含离线教程与 FAQ，并支持导出本地日志用于问题排查。
- **FR-025**: 系统 MUST 在关键失败场景提供可理解的错误原因与可执行的解决建议（如素材不合规、空间不足、硬件能力不足等）。

### Key Entities *(include if feature involves data)*

- **文案（Script）**: 用户输入/导入的口播内容，包含原文与优化版的版本关系。
- **音色（Voice）**: 可被试听与选用的语音配置项（内置或用户自定义），包含可调节参数与收藏状态。
- **数字人（Avatar/Digital Human）**: 可被预览与选用的口播形象（官方、图片生成或用户自定义），包含来源、预览素材与收藏状态。
- **制作配置（Project Config）**: 单条/批量生成时的完整配置快照，用于“重新编辑”。
- **作品（Work）**: 已生成的视频及其元信息（名称、路径、缩略图、创建时间等）。
- **存储配置（Storage Settings）**: 与模型/素材/作品本地路径相关的设置与约束。
- **硬件等级（Hardware Tier）**: 启动检测后得到的能力等级，用于限制批量条数与素材大小上限等规则。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 新手用户在首次打开软件后，可在 1 分钟内完成“进入单条制作 → 输入文案 → 选择音色与数字人 → 发起生成”的操作闭环。
- **SC-002**: 软件启动后，硬件检测与等级展示在 10 秒内完成。
- **SC-003**: 文案优化操作在 3 秒内返回结果；超过 3 秒时提供明确的加载反馈与可取消/重试路径。
- **SC-004**: 音色试听在用户触发后 1 秒内开始播放或给出明确失败原因。
- **SC-005**: 图片生成数字人耗时满足分级目标：入门档 ≤ 60 秒、推荐档 ≤ 30 秒、专业档 ≤ 15 秒。
- **SC-006**: 生成的视频输出为 1080P，并且口型同步误差 ≤ 0.1 秒（以抽样评测方式验证）。
- **SC-007**: 单条视频生成过程具备可理解的阶段进度提示，并且暂停/继续/取消操作在 1 秒内产生可感知响应。
- **SC-008**: 批量制作支持在专业档单次生成 100 条文案；在入门档单次生成条数不超过 10 条，并对超限行为给出明确提示。
- **SC-009**: 作品库在作品数量达到 500 条时，用户执行名称搜索或排序操作在 1 秒内返回结果。
- **SC-010**: 删除作品或清理缓存后，系统可展示释放空间结果，并确保用户作品不被误删。

## Assumptions

- 默认“全程本地离线生成”指：除用户主动进行更新/下载素材等动作外，生成流程不要求联网；用户内容不上传云端。
- 输出视频默认无水印。
- 批量制作“按硬件等级限制单次条数/素材大小”属于明确的产品规则，等级阈值与限制项可在后续规划阶段细化为可配置的验收表。
- PRD 源文件在“单张图片生成数字人技术方案”处出现截断，本文档在不引入实现细节的前提下，对缺失内容以“可验收的用户价值与边界规则”方式补全。
