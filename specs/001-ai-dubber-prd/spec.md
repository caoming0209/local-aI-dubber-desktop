## User Scenarios & Testing *(mandatory)*

### User Story 1 - 单条生成口播视频（离线、1080P/720P、含字幕）(Priority: P1)

用户在无网络环境下，通过上传数字人图片、输入/粘贴文案、选择声音模板，生成一条可用的口播视频；生成过程中获得清晰的进度与资源状态反馈；生成完成后可直接定位输出文件。

**Why this priority**: 单条生成是产品的最小可用闭环，决定用户能否“马上产出”。

**Independent Test**: 仅实现该故事即可独立验证端到端闭环：准备素材 → 发起生成 → 获取输出文件（视频+字幕） → 打开输出目录。

**Acceptance Scenarios**:

1. **Given** 用户处于无网络环境且进入单条生成页面，**When** 用户提供合规的图片、文案与声音模板并点击生成，**Then** 系统开始生成并持续提供阶段进度、总体进度与资源状态展示，最终输出视频文件并提供“查看输出目录”入口。
2. **Given** 用户选择输出分辨率为 720P，**When** 用户发起生成，**Then** 输出的视频分辨率为 720P 且页面清晰显示用户的分辨率选择与其含义（例如“更稳定/更快”）。
3. **Given** 用户文案超过单段字数上限，**When** 用户发起生成，**Then** 系统对文案做自动拆分并在最终结果侧保证内容连贯，用户获得一条连续可用的视频。
4. **Given** 生成进行中，**When** 用户点击取消并在确认弹窗中确认，**Then** 系统停止生成、清理临时产物并提示“已取消”，且用户可立即再次发起新的生成。
5. **Given** 视频生成完成但字幕生成出现失败，**When** 系统完成重试后仍失败，**Then** 系统仍输出视频且明确提示字幕未生成但不影响视频结果。
6. **Given** 用户选择某声音模板并进入语速/音量/情感（风格）设置，**When** 用户不做任何修改直接生成，**Then** 系统默认选中并使用“中性/自然（推荐）”作为最佳参数集。
7. **Given** 用户在生成前调整了语速/音量并选择了非默认情感/风格，**When** 用户发起生成，**Then** 本次任务按用户调整后的参数执行，且界面配置摘要能清晰展示已生效的关键参数（避免误以为仍是默认推荐项）。

### User Story 2 - 本地声音克隆与声音模板管理（可复用、可删除）(Priority: P2)

用户上传本地音频，通过本地处理提取可复用的声音特征并保存为“声音模板”；用户可在生成时选择不同模板以保持人设一致；模板可删除且删除前需确认。

**Why this priority**: 声音一致性是口播视频的核心体验要素之一，也是用户选择本产品的重要原因。

**Independent Test**: 仅实现该故事即可独立验证“上传音频 → 提取 → 命名保存 → 列表选择/删除”的完整闭环。

**Acceptance Scenarios**:

1. **Given** 用户上传了合规音频文件，**When** 用户点击“提取声音”，**Then** 系统展示进行中的状态反馈，提取成功后要求用户输入模板名称并保存。
2. **Given** 用户上传不合规音频（格式不支持、时长不足（<30秒）或文件过大（>100MB）），**When** 用户尝试提取，**Then** 系统阻止提取并明确提示原因与如何修正。
3. **Given** 用户已有多个声音模板，**When** 用户在模板列表中选择其中一个，**Then** 选中状态清晰可见且该模板可被用于后续单条/批量生成。
4. **Given** 用户尝试删除模板，**When** 用户在确认弹窗中确认删除，**Then** 模板从列表中移除且不可恢复。

### User Story 3 - 批量生成（≤30条、串行执行、断点续传、可监控）(Priority: P3)

用户通过 TXT（自动识别 UTF-8/UTF-8 BOM/GBK）或 Excel（仅 .xlsx）导入多条文案（单批不超过安全上限），统一配置图片、声音模板与分辨率后发起批量生成；生成严格串行执行；任意条目失败不影响后续条目；异常退出后重启可选择继续未完成任务（续跑输出目录固定为批次创建时目录）。

**Why this priority**: 批量生产是目标用户的高频场景，“断点续传 + 任务监控”是稳定落地的关键兜底体验。

**Independent Test**: 仅实现该故事即可在 3 条文案规模下验证：导入 → 串行生成 → 显示每条状态 → 模拟异常退出 → 重启继续未完成条目。

**Acceptance Scenarios**:

1. **Given** 用户成功导入 N 条文案且 N ≤ 30，**When** 用户开始批量生成，**Then** 系统串行执行并为每条展示等待/执行中/成功/失败状态与进度。
2. **Given** 用户导入超过 30 条文案，**When** 导入完成，**Then** 系统明确提示超出安全上限并仅保留前 30 条用于本次批量。
3. **Given** 批量生成过程中某一条任务失败，**When** 失败发生，**Then** 该条标记失败并记录可展示的失败原因，后续条目继续执行，最终汇总成功/失败数量；并允许用户对失败条目发起单独重试。
4. **Given** 批量生成过程中发生异常退出（崩溃/断电），**When** 用户重启软件，**Then** 系统提示存在未完成任务并允许用户“一键继续”或“放弃并清空”，选择继续后从未完成条目继续；异常退出时正在执行的条目视为未完成并在恢复时从头重跑；已完成条目不重复生成；断点续传时输出目录固定为该批次创建时的输出目录。
5. **Given** 批量生成进行中，**When** 用户点击“取消批量”并在确认弹窗中确认，**Then** 系统停止后续条目并清理临时文件；已完成条目产物保留；正在执行条目尽力中止并标记为失败/未完成；用户可重新发起新的批量生成。

---

### Edge Cases

- 离线环境下启动与使用：除用户主动触发需要联网的动作外，不应出现网络相关阻断提示。
- 输入质量不足：图片分辨率不足、音频时长不足、文案为空/全空行等，应提供可理解提示并引导用户修正。
- 资源不足：显存压力、磁盘空间不足时，应提供预警与可执行建议，且不遮挡核心操作。
- 字幕异常：字幕失败不应阻断视频产出，需给出明确状态。
- 取消/放弃：取消单条或放弃批量后，应清理临时文件并保持可再次执行。
- 并发生成：生成任务进行中时禁止启动新任务；"开始生成"按钮禁用并给出明确提示。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 支持离线运行：断网后核心生成能力可正常使用，且不出现与联网相关的阻断提示。
- **FR-001a**: 安装包 MUST 内置核心推理模型（TTS CosyVoice3-0.5B + 口型同步 Wav2Lip），确保用户安装后无需联网即可使用生成功能。
- **FR-002**: 系统 MUST 保护用户隐私：文案、图片、音频、生成视频不收集、不上传。

- **FR-010**: 系统 MUST 支持上传数字人图片（JPG/PNG）并提供预览、删除与重新上传；上传后 MUST 自动 resize 为 512×512（使用中心裁切保持主体居中）作为后续生成输入。
- **FR-011**: 系统 MUST 对不合规图片给出明确提示（至少包含：格式不支持、分辨率低于 256×256）。

- **FR-020**: 系统 MUST 支持输入/粘贴口播文案并展示字数状态。
- **FR-020a**: 系统 MUST 限制单条任务文案总字数不超过 3000 字；超限时 MUST 提示用户缩减文案或拆分为多条任务。
- **FR-021**: 系统 MUST 在文案超过单段上限（120 字/段）时自动拆分并在结果侧保持内容连贯。

- **FR-030**: 系统 MUST 支持上传音频（MP3/WAV）作为声音克隆输入。
- **FR-031**: 系统 MUST 对音频输入设置并验证约束（至少包含：最小时长 30 秒、最大文件大小 100MB、格式限制），并对不合规输入提供可理解的失败原因与修正建议。
- **FR-032**: 系统 MUST 支持将提取结果保存为可复用的声音模板，并在模板列表中展示模板基础信息（至少包含：名称、创建时间、来源音频时长）。
- **FR-032a**: 声音模板名称 MUST 唯一；保存时若名称已存在，MUST 阻止保存并提示用户修改名称。
- **FR-033**: 系统 MUST 支持选择声音模板（用户自建 voice_templates）用于单条/批量生成。V1.3 范围内仅支持用户创建的声音模板；预置音色模型（voice_models）的选择在后续版本实现。
- **FR-033a**: 系统 MUST 支持在生成时对所选声音模板进行参数调节（至少包含：语速、音量、情感/风格）；并提供“推荐/最佳”默认选中项（默认：中性/自然）。
- **FR-033b**: 系统 MUST 将“推荐/最佳”作为默认参数集应用到新建任务；用户修改参数后，修改结果 MUST 仅影响当前任务配置（单条/批量均以当前配置为准）。
- **FR-034**: 系统 MUST 支持删除声音模板；删除前 MUST 二次确认；删除后 MUST 不可恢复。

- **FR-040**: 系统 MUST 支持单条视频生成，且默认输出 1080P；并允许用户切换为 720P。
- **FR-041**: 系统 MUST 清晰展示当前分辨率选择，并向用户解释 1080P/720P 的差异（例如“推荐/更稳定”）。

- **FR-050**: 系统 MUST 在单条生成过程中展示进度（至少包含：总体进度与当前阶段描述）。
- **FR-051**: 系统 MUST 支持取消单条生成；取消需确认；取消后 MUST 清理临时文件并提示取消结果。

- **FR-060**: 系统 MUST 默认同时输出外置字幕文件（.srt 格式）与视频内嵌字幕（硬字幕）。
- **FR-061**: 系统 MUST 在字幕生成失败时自动重试一次；重试仍失败时 MUST 提示字幕失败但不阻断视频产出。
- **FR-062**: 系统 MUST 允许用户查看字幕文件并复制字幕文本（如复制成功需有明确反馈）。

- **FR-070**: 系统 MUST 支持批量导入文案（TXT/Excel(.xlsx)）。TXT 编码 MUST 自动识别 UTF-8/UTF-8 BOM/GBK。导入后允许用户预览、编辑、删除条目与清空。
- **FR-071**: 系统 MUST 限制单批任务规模不超过 30 条；超过时 MUST 明确提示并限制本次任务规模。
- **FR-072**: 系统 MUST 严格串行执行批量任务。
- **FR-073**: 系统 MUST 在批量生成时展示每条任务状态（等待/执行中/成功/失败）与整体汇总。
- **FR-074**: 系统 MUST 支持批量断点续传（按条目续跑）：异常退出后重启可选择继续；继续后从未完成条目开始执行；异常退出时正在执行的条目视为未完成并在恢复时从头重跑；已完成条目不重复生成。断点续传时输出目录 MUST 固定使用批次创建时的输出目录。
- **FR-075**: 系统 MUST 支持对批量任务中失败条目发起单独重试；重试仅针对失败条目重新生成且不影响其它条目。
- **FR-076**: 系统 MUST 支持取消整个批量任务；取消后停止后续条目并清理临时文件；已完成条目产物保留；取消时正在执行的条目尽力中止并视为失败/未完成。

- **FR-080**: 系统 MUST 提供资源状态展示，至少包含 CPU/内存占用与 GPU 显存占用（仅 CPU 推理时显存展示为 N/A（未使用 GPU）），并在生成过程中以不低于每 2 秒 1 次的频率更新显示。
- **FR-081**: 系统 MUST 在资源压力达到预警阈值时提示用户：显存 ≥ 90%、内存 ≥ 90%、磁盘剩余 < 1GB。提示不遮挡核心操作区。
- **FR-082**: 系统 MUST 在无法安全继续时自动暂停当前生成任务，弹出非阻断提示告知资源超限原因，并建议用户关闭其他程序、缩小批量规模、或在设置中切换推理模式后手动恢复；暂停期间已完成产物 MUST 保留。


- **FR-085**: 系统 MUST 阻止并发生成：当任一生成任务（单条或批量）正在执行时，禁用"开始生成"按钮并提示用户需等待当前任务完成或取消后才可发起新任务。

- **FR-090**: 系统 MUST 在生成完成后提供可直接打开输出目录的入口，并允许用户复制输出路径。

### Functional Requirement Acceptance Criteria

- **FR-001**: 断网条件下，用户仍可完成单条/批量生成全流程；流程中不出现”需要联网才可继续”的阻断提示。
- **FR-001a**: 安装完成后无需额外联网下载模型即可使用生成功能；核心模型（TTS + 口型同步）随安装包内置。
- **FR-002**: 核心生成流程不要求用户执行任何上传/同步步骤；用户产物可在本地输出目录被直接访问与使用。

- **FR-010**: 用户可上传 JPG/PNG 并在界面预览；用户可移除并重新上传后继续流程；上传后系统自动 resize 为 512×512（中心裁切）用于生成。
- **FR-011**: 图片格式不支持或分辨率低于 256×256 时，系统阻止进入生成并提示可理解原因。

- **FR-020**: 用户输入/粘贴文案时，界面持续展示字数状态（包含超限提醒）。
- **FR-020a**: 文案总字数超过 3000 字时，系统阻止生成并提示用户缩减文案或拆分为多条任务。
- **FR-021**: 文案超过单段上限时，系统自动拆分并在结果侧保持连续可用，不要求用户手动拆分。

- **FR-030**: 用户可选择并上传 MP3/WAV 音频作为克隆输入。
- **FR-031**: 音频不合规（格式/时长<30秒/大小>100MB）时，系统阻止提取并提示原因与修正建议。
- **FR-032**: 提取成功后用户可命名并保存模板；模板列表至少展示名称、创建时间、来源音频时长。
- **FR-032a**: 模板名称必须唯一；保存时若名称已存在，系统阻止保存并提示用户修改名称。
- **FR-033**: 用户可在单条/批量生成时选择已保存的声音模板（V1.3 仅支持用户自建 voice_templates，不含预置 voice_models）。
- **FR-033a**: 用户可在生成时调节语速、音量、情感/风格等参数，且默认选中“推荐/最佳”项（默认：中性/自然）。
- **FR-033b**: “推荐/最佳”参数集 MUST 为新建任务的默认值；用户修改参数后 MUST 仅影响当前任务配置（单条/批量均以当前配置为准）。
- **FR-033c**:
  - 语速、音量 MUST 支持可视化调节（如滑杆或数值输入）；
  - 情感/风格 MUST 以可枚举选项提供（单选）；
  - “中性/自然（推荐）” MUST 明确标注为推荐项并默认选中。
- **FR-033d**: 系统 MUST 在单条/批量生成的配置摘要中展示已选择的情感/风格与语速/音量关键值，避免用户误以为仍是默认“推荐/最佳”。
- **FR-034**: 删除模板需二次确认；确认后模板从列表移除且不可恢复。

- **FR-040**: 单条生成默认输出 1080P，且用户可切换 720P 并输出对应分辨率视频。
- **FR-041**: 分辨率选择在界面清晰可见，且包含对 1080P/720P 差异的用户可理解说明。

- **FR-050**: 单条生成过程中展示总体进度与当前阶段描述，且随流程推进持续更新。
- **FR-051**: 用户可取消单条生成并需确认；取消后提示“已取消”，且可立即发起新生成。

- **FR-060**: 生成完成时，输出包含外置字幕文件（.srt 格式），且视频默认内嵌字幕。
- **FR-061**: 字幕失败自动重试一次；重试仍失败时，仍输出视频并明确提示“字幕失败但不影响视频”。
- **FR-062**: 用户可查看字幕文件并复制字幕文本；复制成功有明确反馈。

- **FR-070**: 用户可从 TXT（自动识别 UTF-8/UTF-8 BOM/GBK）或 Excel（仅 .xlsx）导入文案；导入后可预览、编辑、删除条目与清空。
- **FR-071**: 导入条目数超过 30 时，系统明确提示超限并仅保留前 30 条。
- **FR-072**: 批量生成严格串行：同一时间仅有一个条目处于执行中。
- **FR-073**: 批量生成展示每条状态（等待/执行中/成功/失败）与整体汇总。
- **FR-074**: 异常退出后重启，系统提示未完成任务并允许继续；继续后从未完成条目开始执行；异常退出时正在执行的条目视为未完成并在恢复时从头重跑；已完成条目不重复生成；断点续传时输出目录固定为批次创建时的输出目录。
- **FR-075**: 用户可对失败条目发起单独重试；重试仅针对失败条目重新生成且不影响其它条目。

- **FR-080**: 生成过程中展示 CPU/内存占用与 GPU 显存占用等资源状态（仅 CPU 推理时显存展示为 N/A（未使用 GPU）），且更新频率不低于每 2 秒 1 次。
- **FR-081**: 资源压力达到预警阈值（显存 ≥ 90%、内存 ≥ 90%、磁盘剩余 < 1GB）时提示用户，且提示不遮挡核心操作区。
- **FR-082**: 资源超限（如显存/内存不足、磁盘空间耗尽）时，系统自动暂停当前任务并弹出非阻断提示，告知原因与建议（关闭其他程序/缩小批量/切换推理模式等）；用户确认后可手动恢复；暂停期间已完成产物保留。

- **FR-083**: 推理模式在 UI 上 MUST 默认使用”自动”；用户仅在设置页（高级）可强制选择 CPU 或 GPU。

- **FR-085**: 任一生成任务（单条或批量）执行中时，”开始生成”按钮禁用且显示提示；用户需等待完成或取消当前任务后方可发起新生成。

- **FR-090**: 生成完成后提供”打开输出目录”入口并允许复制输出路径。

### Key Entities *(include if feature involves data)*

- **声音模板（Voice Template）**: 用户可复用的声音资产；关键属性包括名称（唯一）、创建时间、来源音频信息、可用于生成的声音特征数据、可用状态。
- **单条生成任务（Single Job）**: 一次单条生成的执行单元；关键属性包括输入素材引用、分辨率选择、阶段进度、开始/结束时间、输出产物引用与失败原因（如有）。
- **批量生成任务（Batch Job）**: 由多个条目组成的串行任务集合；关键属性包括条目列表、统一配置、总体状态与汇总结果。
- **任务快照（Batch Snapshot）**: 支撑断点续传的任务记录；关键属性包括每条条目的状态、当前执行位置、最后更新时间。
- **输出产物（Outputs）**: 用户可直接使用的文件集合；至少包含视频文件与字幕文件，并可包含任务元数据与日志。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 在无网络环境下，用户可完成单条生成全流程并获得可用视频与字幕文件；首次尝试成功率 ≥ 90%（基于可控的可用性测试样本）。
- **SC-002**: 在目标硬件（8GB 显存 NVIDIA 显卡）上，批量 30 条任务可完整跑通且不出现由显存溢出导致的应用崩溃。
- **SC-003**: 批量生成异常退出后，用户重启软件可在 1 次明确操作内完成“继续未完成任务”，且已完成条目不重复生成。
- **SC-004**: 在目标硬件上，单条 1080P 生成耗时中位数 ≤ 3 分钟（以标准样例素材集测试）。
- **SC-005**: 资源预警与异常提示可被用户理解并采取行动：在可用性测试中，≥ 80% 的用户能依据提示完成一次有效的自助处理（例如改用更稳定的输出选项或缩小批量规模）。

## Assumptions

- 本规格覆盖 PRD V1.3 中的 P0/P1 核心闭环：离线单条生成、离线批量生成、声音克隆模板、字幕、资源监控与稳定性兜底。
- 2K/4K 输出、字幕编辑、高级插帧等属于后续优化或高配版本能力，不在本次范围内。
- Excel（仅 .xlsx）导入以“每行一条文案”为原则；若出现多列，默认取第一列文本作为文案内容。
- 批量生成默认统一使用一张数字人图片与一个声音模板；若未来支持逐条配置，需另行补充规格。
- 作品库（浏览/搜索/播放/删除）为已有模块，本迭代不做功能性变更；但 MUST 确保该模块在离线环境下可正常运行（FR-001 覆盖范围确认）。
- 授权与激活模块（激活码验证、解绑、试用扣减）不在 V1.3 核心闭环范围内；`contracts/license.md` 保留为接口草案，待后续迭代落地。开发模式下（NODE_ENV=development）授权检查跳过。
